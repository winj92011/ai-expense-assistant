# 飞书集成路线

## v0.1 必接能力

1. 企业自建应用
2. 网页应用主页
3. OAuth 免登录
4. 通讯录读取
5. 机器人消息
6. 交互式消息卡片

## 免登录流程

1. 员工从飞书工作台打开报销应用
2. 前端获取飞书授权码
3. 后端用授权码换取用户 `open_id`
4. 后端按 `open_id` 匹配内部员工
5. 返回业务会话

## 消息卡片动作

建议第一版只保留三类动作：

- 查看详情
- 同意
- 驳回

审批动作必须回调到自有后端，由后端写入审批记录和审计日志，再同步页面状态。

## 权限建议

- 获取用户身份
- 获取通讯录基础信息
- 发送机器人消息
- 发送交互式消息卡片
- 读取审批实例，若第一版选择同步飞书原生出差审批

## 产品边界

飞书负责入口、身份、通知和协同；报销规则、AI 推断、审批状态、票据附件和财务台账由自有系统负责。
