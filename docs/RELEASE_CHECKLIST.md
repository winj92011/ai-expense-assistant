# 发布前检查清单

这份清单用于把本地原型能力同步到线上前做最后确认。当前项目仍是产品原型，发布目标是让演示地址稳定可用，不是一次性完成企业级后端。

## 1. 本地自动验证

在项目目录运行：

```bash
npm test
```

通过标准：

- 契约校验通过，代表 API 文档、数据模型、环境变量文档、README 入口和 prototype mock API 运行时响应没有断裂。
- 所有 Playwright E2E 通过。
- `smoke.spec.js` 通过，代表 `?smoke=1` 页面内冒烟入口可用。
- 没有新增未解释的失败截图或 trace。

## 2. 功能确认

发布前至少确认这些能力仍可用：

- 员工提交报销草稿。
- 主管审批同意。
- 财务复核、填写审计说明、生成凭证包。
- 出纳付款，入账状态变为已入账。
- 飞书、钉钉、浏览器测试平台切换。
- 中英文关键文案切换。
- 角色权限模拟。
- 浏览器本地暂存保存、恢复、清空。
- 数据对象预览和 JSON 导出。

## 3. 线上 Smoke Test

部署后打开：

```text
https://ai-expense-assistant.vercel.app/?smoke=1
```

通过标准：

- 页面出现“原型流程验证通过”。
- 面板中包含“员工提交”“财务复核后进入付款”“出纳付款闭环”“财务归档对象生成”。

## 4. 人工抽查

每轮中等改动后做一次 2-5 分钟人工 smoke：

1. 打开线上首页。
2. 切换企业平台、语言和角色。
3. 走一遍员工提交、主管同意、财务复核、出纳付款。
4. 查看凭证包、财务筛选、本地暂存、数据对象预览。

## 5. 不在本次发布范围

- 真实企业数据库。
- 真实附件对象存储。
- 飞书/钉钉消息卡片线上回调。
- 正式登录权限系统。
- 财务系统自动入账。

这些属于后续企业级后端接入阶段。
## 发布成熟度

当前发布目标是“稳定演示版”，不是“企业生产版”。发布前应确认 `docs/PILOT_READINESS.md` 中的能力边界仍然准确，尤其是真实数据库、正式权限、附件存储和真实飞书/钉钉回调是否仍属于后续范围。

如果发布内容涉及真实持久化，应额外对照 `docs/PERSISTENCE_PLAN.md` 检查数据库、API、审计日志和恢复流程。
