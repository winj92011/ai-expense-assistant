# AI 报销助理原型演示指南

这份指南用于演示当前原型能力。当前阶段仍是产品原型，不假装已经具备完整企业级后端；本地暂存、角色切换和平台载荷都用于演示和验证后续接入方向。

## 演示主线

1. 打开首页，先看企业平台、语言、角色和本地暂存入口。
2. 选择“浏览器测试”平台，说明飞书和钉钉的适配层已经预留。
3. 切换“原型角色视角”，展示员工、主管、财务、出纳、管理员看到的入口不同。
4. 上传或模拟票据，生成报销草稿。
5. 对“需补充说明”的费用填写员工说明。
6. 员工确认并提交。
7. 切到主管视角，查看员工补充说明并同意。
8. 切到财务视角，查看归档号、入账状态、审计说明、凭证包和财务筛选。
9. 填写复核说明，标记已复核，下载凭证包。
10. 确认付款，查看入账状态变为已入账。
11. 查看“数据对象预览”，说明未来后端需要保存的对象结构。
12. 导出 JSON 或 CSV，用于解释数据库、附件和财务归档的后续接入。

## 推荐讲法

- 员工侧：AI 先生成可编辑草稿，异常只提示，不强行阻断提交。
- 主管侧：重点看异常说明、金额、行程合理性和审批链。
- 财务侧：重点看归档号、凭证包、审计说明、入账状态和导出台账。
- 平台侧：飞书、钉钉、浏览器测试共用一套业务流，只替换身份、审批和消息卡片适配层。
- 数据侧：当前页面已经能展示未来落库对象，字段草案见 `docs/DATA_MODEL.md`，但还没有接真实数据库和附件存储。

## 自动验证

完整自动验证：

```bash
npm test
```

当前覆盖：

- `finance-pack.spec.js`：财务凭证包、筛选、复核、付款闭环
- `platform-adapter.spec.js`：飞书、钉钉、浏览器测试载荷
- `role-permissions.spec.js`：角色视角和导航权限
- `i18n-preview.spec.js`：中英文关键文案切换
- `exception-notes.spec.js`：员工补充说明流转到主管和财务
- `prototype-persistence.spec.js`：浏览器本地暂存保存、恢复、清空
- `data-model-preview.spec.js`：数据对象预览和 JSON 导出
- `smoke.spec.js`：页面内 `?smoke=1` 冒烟验证入口
- `system-readiness.spec.js`：AI 调试页展示环境、mock API 和验证入口状态

## 人工 Smoke Test

每累计一轮中等改动后，做一次 2-5 分钟人工检查：

1. 打开线上页面。
2. 切换企业平台和语言，确认关键文案没有明显错位。
3. 切换角色，确认导航权限符合预期。
4. 走一遍员工提交、主管同意、财务复核、出纳付款。
5. 查看凭证包、财务筛选、本地暂存和数据对象预览。

高风险改动需要单独人工验证：AI 识别接口、提交链路、权限逻辑、财务状态、数据持久化。

## 暂不承诺

- 不承诺已有真实企业数据库。
- 不承诺已有真实附件存储。
- 不承诺飞书/钉钉消息卡片已经线上回调闭环。
- 不承诺当前角色切换等于正式权限系统。

这些能力已经在原型里留出结构，后续可以逐步接入真实后端。
## 当前演示定位

当前版本适合演示完整产品故事：AI 识别、员工提交、主管审批、财务复核、凭证包、付款闭环、平台适配、身份上下文和数据对象预览。

需要明确说明：当前约为 55% 产品化成熟度，适合演示，不等于真实企业生产系统。试点路线见 `docs/PILOT_READINESS.md`。
